RG to NFA Instructions

1. Input regular grammar (example: S → aS | ε )
2. Each sentence is separated by line break (Enter or "\n")
3. State must be one uppercase alphabet [A-Z]
4. Transition variable must be number [0-9] or lowercase alphabet [a-z]
5. First sentence will be an initial state (entry point)
6. Sentence arrow is denoted by an arrow ("→") ex. S -> aS (❌), S → aS (✔️)
7. Multiple transition word is denoted by a pipe ("|") ex. aS | bA | abB
8. No space for each transition word ex. S → a S (❌), S → aS (✔️)
9. Word parsing will stop when encountered an uppercase letter (State), subsequent
    letter after an uppercase letter will be ignored. ex. aSaaa (will be read as aS ), Ca (will be parse as C)

NFA to RG Instructions

1. Click the ‘Insert ε’ button to insert the epsilon symbol into the "String Variables" and String to Check" input field.
2. Generate NFA Table
- Input the variables, states, start state, and final states in the corresponding input fields.
- Click the ‘Generate Transition Table’ button to generate an HTML table representing the NFA.
3. Convert NFA to Regular Grammar
- Update the corresponding input field with the transition data.
- Click the ‘Convert NFA to RG’ button to convert the NFA to Regular Grammar.
4. Input the strings you want to check into the "String to Check" input field (left column).
- Click the ‘Check String’ button to check the strings against the NFA.
- The results will be displayed in the right column(Yes = accepted, No = rejected).
5. Draw NFA Diagram
- Click the ‘Draw NFA’ button to generate a canvas 2d diagram representing the NFA.
